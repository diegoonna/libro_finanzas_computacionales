4 Introducción

8 Problemática Financiera: Valuación de Bonos

8.1 Valor Presente Neto (VPN)

La fórmula fundamental de valuación por flujos descontados:

\[VPN = \sum_{t=1}^{n} \frac{CF_t}{(1 + r)^t}\]

Donde \(CF_t\) es el flujo de fondos en el período \(t\) y \(r\) es la tasa de descuento.

# TODO: Implementar VPN con bucle for
flujos = [80, 80, 80, 80, 1080]
yield_mercado = 0.10

vpn = 0
print("Desglose del Valor Presente:")
print("-" * 45)

for t, cf in enumerate(flujos, start=1):
    factor_descuento = 1 / (1 + yield_mercado) ** t

9 Bucles while y Búsqueda de la TIR

El bucle while se ejecuta mientras una condición sea verdadera. Lo usaremos para buscar la Tasa Interna de Retorno (TIR) por tanteo.

9.1 TIR por Búsqueda Incremental

La TIR es la tasa \(r^*\) que hace que el VPN sea igual al precio de mercado:

\[P_{mercado} = \sum_{t=1}^{n} \frac{CF_t}{(1 + r^*)^t}\]

# TODO: Implementar búsqueda de TIR por tanteo (método ingenuo)
flujos = [-924.18, 80, 80, 80, 80, 1080]  # Flujo 0 = precio pagado (negativo)
# El primer flujo es la inversión inicial (negativa)

tasa_prueba = 0.01   # Empezamos en 1%
paso = 0.0001         # Incremento
max_iteraciones = 100_000

iteraciones = 0
while tasa_prueba < 1.0 and iteraciones < max_iteraciones:
    vpn = 0
    for t, cf in enumerate(flujos):
        vpn += cf / (1 + tasa_prueba) ** t
    
    if abs(vpn) < 0.01:  # Tolerancia
        break
    
    tasa_prueba += paso
    iteraciones += 1

print(f"TIR encontrada: {tasa_prueba:.4%}")
print(f"VPN residual: ${vpn:.4f}")
print(f"Iteraciones: {iteraciones:,}")
TIR encontrada: 10.0000%
VPN residual: $0.0043
Iteraciones: 900
AdvertenciaLimitaciones del Método por Tanteo

Este método es ineficiente (O(n) iteraciones). En la Semana 3 veremos métodos más eficientes, y en módulos posteriores usaremos scipy.optimize que implementa algoritmos como Newton-Raphson.

# Bucles `while` y Búsqueda de la TIR

El bucle `while` se ejecuta mientras una condición sea verdadera. Lo usaremos para buscar la **Tasa Interna de Retorno (TIR)** por tanteo.

## TIR por Búsqueda Incremental

La TIR es la tasa $r^*$ que hace que el VPN sea igual al precio de mercado:

$$P_{mercado} = \sum_{t=1}^{n} \frac{CF_t}{(1 + r^*)^t}$$

```{python}
#| label: tir-tanteo
# TODO: Implementar búsqueda de TIR por tanteo (método ingenuo)
flujos = [-924.18, 80, 80, 80, 80, 1080]  # Flujo 0 = precio pagado (negativo)
# El primer flujo es la inversión inicial (negativa)

tasa_prueba = 0.01   # Empezamos en 1%
paso = 0.0001         # Incremento
max_iteraciones = 100_000

iteraciones = 0
while tasa_prueba < 1.0 and iteraciones < max_iteraciones:
    vpn = 0
    for t, cf in enumerate(flujos):
        vpn += cf / (1 + tasa_prueba) ** t
    
    if abs(vpn) < 0.01:  # Tolerancia
        break
    
    tasa_prueba += paso
    iteraciones += 1

print(f"TIR encontrada: {tasa_prueba:.4%}")
print(f"VPN residual: ${vpn:.4f}")
print(f"Iteraciones: {iteraciones:,}")
```

::: {.callout-warning title="Limitaciones del Método por Tanteo"}
Este método es **ineficiente** (O(n) iteraciones). En la Semana 3 veremos métodos más eficientes, y en módulos posteriores usaremos `scipy.optimize` que implementa algoritmos como Newton-Raphson.
:::

# Complejidad Algorítmica (Introducción)

| Notación | Nombre | Ejemplo |
|----------|--------|---------|
| $O(1)$ | Constante | Acceder a un elemento por índice |
| $O(n)$ | Lineal | Recorrer una lista |
| $O(n^2)$ | Cuadrática | Bucle anidado (ej: correlaciones) |

```{python}
#| label: complejidad
# O(1) - Acceso directo
flujos = [80, 80, 80, 80, 1080]

# O(n) - Recorrido lineal
suma = 0
for f in flujos:  # n operaciones
    suma += f
print(f"Suma O(n): {suma}")

# O(n²) - Ejemplo: comparar todos los pares
n = len(flujos)
comparaciones = 0
for i in range(n):
    for j in range(i + 1, n):
        comparaciones += 1
print(f"Comparaciones O(n²): {comparaciones}")
```
